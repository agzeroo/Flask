# 웹 이해
- 포지션
    - 클라이언트
        - 웹을 사용하는 유저
        - 유저의 입장에서 개발
        - 행동
            - 브라우저 오픈(크롬, 사파리, 에지,...)
            - 주소창에 URL 입력( www.naver.com, .... 127..(X:) )
            - URL을 입력후 엔터 -> request가 발생(요청을 서버로 전송)
    - 서버
        - 서비스를 제공하는 기업(개인)
        - 절대로 다운되면 않된다. (서비스는 중단되면 않된다)
        - 365일 풀 가동
        - 클라이언트로부터 요청을 받으면
            - URL 분석
            - URL 을 처리하는 함수에 호출해서 요청을 던진다 : 라우팅
            - 함수는 호출되서 업무를 수행한다 : 비즈니스 로직 처리
            - 로직 처리(예측)후 결과를 가지고 응답처리(response)
                - html 응답 -> 화면이 존재한다
                - xml / json 응답 -> 화면이 존재하지 않는다(미들웨어)
    - 데이터베이스
        - 필요에 따라 존재할수도, 없을수도 있다
        - 데이터를 저장하는 저장소
        - 비즈니스 로직의 목적상 (로그인 경우), 회원정보가 디비에 저장되
          어 있으므로, 디비에 쿼리를 던져서 결과를 받아와야한다
          - 서비스 제공 -> 사용자가 이용 -> 그 데이터를 저장(동의)
          - 이 데이터를 가지고 향후 학습에 재료로 사용 -> 업그레이드
          - 데이터를 수집하기 위해서 무료료 서비스를 제공하는 경우가 많다





- 개발언어
    - 클라이언트
        - HTML(현버전 5)
        - CSS(현버전 3)
        - Javascript(ECMA 2015~2020)
    - 서버
        - Spring / jsp / Servlet / EJB : Java (2000~),  전자정부 프레임웍
        - php                       : php
        - asp / asp.Net / .Net      : C# , MS제품이나 특화된 기업, 여행사, 항공사,
        - nodejs                    : javascript or typescript (2011~)
        - flask / Django            : python (2015~)
                                      딥러닝->파이썬->웹서비스 연동시 언어가 통일        
    - 데이터베이스
        - RDBMS : 오라클,  MSSQL,  MYSQL, 오로라(aws), IBM,..,
        - No-SQL: 몽고디비

- 개발언어
    - 클라이언트
        - HTML : 웹 페이지상에 뼈대 / 구조 담당, 콘텐츠를 가지고 있음
        - CSS : 디자인, 프리젠테이션, 레이아웃, 애니메이션
        - Javascript : 이벤트 처리, 인터렉션(사용자 <-> 웹화면), 통신(네트워크), 화면조작(DOM조작)

    - 서버 
        -flask
            - 마이크로 에디션
            - 유사폼 <-> nodejs
            - 자유도가 높고, 빠르게 구축 가능, 제약이 거의없다.

        -Django
            - 풀 스텍 에디션
            - 유사폼 <-> Spring
